## Successes

1. **Authorization widening done safely** — Story 6.2 widened `GET /api/activities/{id}` from ADMIN+ to VIEWER+ with clear security justification. Auth check moved before DB query for fail-fast. Write endpoints untouched. Non-breaking additive DTO changes.
2. **Semantic token lesson applied proactively** — Story 6.1 code review caught raw Tailwind colors (HIGH issue: `bg-indigo-50` → `bg-primary/5`). Stories 6.2 and 6.3 used semantic tokens from the start. Story 6.2 had zero code review issues — direct evidence of learning compounding within the epic.
3. **Component reuse without modification** — `StaffingIndicator` from Epic 4 reused as-is in 6.2 and 6.3. `InitialsAvatar` used across all 3 stories at different sizes. `formatActivityDate`, `formatTime`, `formatRelativeDate` shared utilities.
4. **Purpose-built DTOs per context** — `MyAssignmentListItem` (personal), `DashboardActivityItem` (dashboard), and widened `ActivityResponse` (detail) each serve specific use cases without modifying the existing admin `ActivityListItem`.
5. **Error/retry states in all 3 stories** — Improvement over Epic 5 where 2/5 stories missed error states. MyAssignmentsSection, ActivityDetailPage, and DashboardUpcomingSection all have proper loading/error/empty states.
6. **Clean auth separation** — Controller handles authorization gates (`CanView()`, `HasActivityAccess()`), service handles data scoping (department filtering by role). Each layer has a single responsibility.
7. **Role-based filtering working correctly** — VIEWER sees all activities, ADMIN sees only assigned departments, OWNER sees everything. Pastor (ADMIN assigned to all departments) gets cross-department visibility per FR43.

## Challenges

1. **EF Core `.AsNoTracking()` cycle error with nested Includes** (6.1) — The Include path `RoleAssignment → ActivityRole → Assignments` caused a cycle when tracking was disabled. The identity map in tracking queries handles cycles automatically. Had to remove `.AsNoTracking()` to match the existing `GetByIdAsync()` pattern. This is a codebase-specific gotcha.
2. **Raw Tailwind colors in initial implementation** (6.1) — HIGH code review issue: `AssignmentCard` and `MyAssignmentsSection` used raw colors (`bg-indigo-50`, `text-indigo-600`). Converted to semantic tokens (`bg-primary/5`, `text-primary`). This was the first authenticated-register component in the epic — the semantic token habit wasn't yet established.
3. **Additional test cases still found in review** (2/3 stories) — Story 6.1: missing OWNER integration test. Story 6.3: missing today-emphasis test, OWNER staffing visibility test, today integration test, avatar URL assertion, View All href assertion. Test coverage gaps persist even as error state handling improves.
4. **UTC date filtering is a systemic limitation** (all 3 stories) — `DateOnly.FromDateTime(DateTime.UtcNow)` may exclude Saturday activities late Saturday evening in Quebec (UTC-5). Documented in all 3 stories as accepted MVP limitation with narrow impact window.
5. **Pre-existing test failures** (6.3) — 3 LiveStatusEndpointTests failures unrelated to the story surfaced during development. Not blocking but adds noise.
6. **App.test.tsx regression** (6.3) — Changing the dashboard greeting from "Tableau de Bord" to "CENTRE DE COMMANDE" broke the existing smoke test. Required test update.

## Key Insights

1. **Code review lessons compound within an epic** — Story 6.1's semantic token correction was proactively applied in 6.2 (zero review issues) and 6.3. The feedback loop works when the same developer implements consecutive stories. This is the ideal learning curve.
2. **`.AsNoTracking()` is unsafe with nested Include chains** — EF Core tracking queries handle cycles via identity map. Read-only queries with complex Include chains should use tracking (no `.AsNoTracking()`) to avoid `InvalidOperationException`. This is counter-intuitive but consistent with the existing codebase pattern.
3. **UTC vs local date filtering is a systemic MVP limitation** — Appears in all 3 stories. Not worth fixing per-story, but worth noting as a single post-MVP item. The timezone-aware approach (`TimeZoneInfo.ConvertTimeFromUtc` with "America/Toronto") is well-understood.
4. **Purpose-built DTOs reduce coupling** — Three contexts (personal, detail, dashboard) warranted three DTO shapes. Modifying the shared `ActivityListItem` would have changed the admin query's performance characteristics and forced every consumer to handle new nullable fields.
5. **Test coverage gaps persist as a review theme** — Despite improved error state handling (Epic 5 feedback applied), additional test cases are still discovered during code review in 2/3 stories. The gap is shifting from "missing states" to "missing edge case coverage."

## Action Items

### Process Improvements

1. **Document `.AsNoTracking()` limitation with nested Includes**
   Owner: Charlie (Dev)
   Success criteria: Codebase convention note: queries with nested `.Include()` chains should NOT use `.AsNoTracking()` — use tracking queries for identity map cycle resolution.
   Rationale: Story 6.1 hit this unexpectedly. Counter-intuitive pattern that should be documented.

2. **Strengthen edge-case test coverage in implementation checklist**
   Owner: Dana (QA) + Charlie (Dev)
   Success criteria: Checklist includes "role-specific test cases" (test each role: VIEWER, ADMIN, OWNER, anonymous) and "boundary date tests" (today, past, future) as mandatory items.
   Rationale: 2/3 stories had missing role-specific or boundary tests caught in review.

### Technical Debt

3. **UTC date filtering → post-MVP timezone-aware filtering**
   Owner: Charlie (Dev)
   Priority: LOW
   Detail: All 3 Epic 6 stories use `DateOnly.FromDateTime(DateTime.UtcNow)` which may differ from Quebec local date (UTC-5) late Saturday evening. Fix: `TimeZoneInfo.ConvertTimeFromUtc(DateTime.UtcNow, TimeZoneInfo.FindSystemTimeZoneById("America/Toronto"))`.

4. **Deferred code review items from Story 6.3**
   - Admin-with-0-departments edge case (M1) — requires test infra change
   - Admin subtitle loading flash (M3) — minor UX polish

5. **All carry-forward items from previous epics (unchanged)**
   See Technical Debt table above. To be addressed in post-retro consolidated action phase.

### Team Agreements

- Semantic tokens only — no raw Tailwind color classes in authenticated components (learned from 6.1 H1)
- `.AsNoTracking()` unsafe with nested Include chains — use tracking queries
- Purpose-built DTOs per context preferred over modifying shared DTOs
- Auth check before DB query (fail-fast pattern)
- Service methods return `AxiosResponse` — unwrap in hook with `.then(res => res.data)`, not in service
- All carry-forward items to be addressed in post-retro consolidated action phase before deployment
