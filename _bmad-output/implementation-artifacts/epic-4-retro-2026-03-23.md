## Successes

1. **Reconcile pattern for child collections** — Implemented in Story 4.1, reused in 4.3 (role roster) and 4.4 (assignments). Compare desired state vs existing state, add/update/remove atomically. Became the backbone of the entire epic.
2. **60% velocity increase** — 8 stories delivered vs 5 in Epic 3, at 100% completion. Largest epic so far.
3. **Test coverage grew 56%** — Frontend tests: 201 → 313 across the epic. Steady growth per story.
4. **`vi.spyOn` pattern fully adopted** — Cleaner than MSW for mutation testing. Consistently used from Story 4.4 onward. Resolved the MSW + FormData + jsdom incompatibility from Epic 3.
5. **Server-computed staffing status with colorblind accessibility** — Shape + color + text (never color alone). WCAG-compliant pattern established in Story 4.7.
6. **Key prop pattern for form reset** — `key={id-${concurrencyToken}}` forces React remount, re-initializing useForm with fresh defaults. Elegant solution to stale form state (Story 4.8).
7. **Force-save 409 loop guard** — Prevents infinite conflict dialog loop on ultra-rare race condition (Story 4.8).
8. **One-commit-per-story discipline improved** — No evidence of mixed-story commits (improvement over Epic 3's 3.4/3.5 issue).

## Challenges

1. **React Hook Form `useFieldArray` edge cases in 5/8 stories** — String `id` conflicts (4.3), `.root` error path (4.3), `defaultValues` not reactive (4.8), `useWatch` returning `undefined` (4.3), private `control._formValues` API (4.3). Each story hit a different edge case — PSI sections helped but didn't aggregate into a reusable reference.
2. **Accessibility/ARIA bugs caught in review, not implementation (3/8 stories)** — Story 4.2: critical `aria-checked` expression always false. Story 4.3: missing labels. Story 4.7: colorblind compliance adjustments. Code review is the last line of defense, but accessibility should shift left.
3. **Silent error swallowing (2/8 stories)** — Story 4.2: missing `onError` on createMutation. Story 4.5: Radix Select empty-string anti-pattern.
4. **Radix jsdom polyfill shared utility still not created** — Carry-forward from Epic 3. Still copy-pasting `hasPointerCapture`, `setPointerCapture`, `releasePointerCapture`, `scrollIntoView` per test file.
5. **Hardcoded critical role detection** — `startsWith("ancien")` and `contains("predicateur")` in `ComputeStaffingStatus`. "Ancienne Musique" false positive. Design issue: string-matching instead of data-driven `isCritical` flag.

---

## Key Insights

1. **PSI sections work for linear transfer, not cumulative knowledge** — Great for story-to-story warnings, but they don't aggregate. A cumulative reference doc (e.g., "RHF gotchas cheat sheet") would compound knowledge instead of rediscovering it linearly.
2. **Code review catches what implementation misses** — Accessibility and error handling are the two biggest blind spots during implementation. Need to shift these left with checklist integration.
3. **Hardcoded business logic is a systemic risk** — The `predicateur`/`ancien` string matching is a symptom. Elisha raised concern that this pattern may be scattered across the codebase. Full audit needed before deployment.
4. **Carry-forward items are intentionally deferred** — Strategy: all retros first, then consolidated action phase. Not a process failure.

## Debug Highlights

| Story | Issue | Resolution |
|---|---|---|
| 4.1 | Npgsql EF Core 10 breaking change: `UseXminAsConcurrencyToken()` removed | `uint Version` property + `IsRowVersion()` configuration |
| 4.1 | Calendar component listed as installed but unavailable | Used native `<input type="date/time">` |
| 4.2 | `aria-checked={selectedId === null && false}` always false | Fixed to `aria-checked={selectedId === null}` |
| 4.2 | createMutation errors silently swallowed | Added `onError` handler for template validation (400) |
| 4.3 | `useFieldArray` injects string `id` conflicting with DB `id` | `.catch(undefined)` on id field in Zod schema |
| 4.3 | `useWatch` for headcount returns `T \| undefined` | Fallback with `?? 1` |
| 4.3 | Private `control._formValues` API | Replaced with `useWatch({ control, name: "roles" })` |
| 4.3 | Field array root error at `.root`, not directly on `errors.roles` | RHF v7 behavior — access `errors.roles?.root` |
| 4.3 | Buggy `templateId` override silently re-applying template roles | Removed override |
| 4.5 | Radix Select ignores empty string values | Sentinel `"none"` value, convert to `null` in `onValueChange` |
| 4.6 | Empty `lastName` on single-word guest names | `InitialsAvatar` handles gracefully |
| 4.7 | `startsWith("ancien")` matches "Ancienne Musique" | Flagged — to be replaced with `isCritical` flag |
| 4.8 | `useForm({ defaultValues })` doesn't react to prop changes | Key prop pattern: `key={id-${concurrencyToken}}` forces remount |
| 4.8 | Force-save could loop on ultra-rare race condition | Check `variables.force` in `onError`, fall back to toast |
