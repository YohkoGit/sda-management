## Successes

1. **Extraordinary velocity** — Complete auth system (Google OAuth + email/password + first-login flow + password reset + session cleanup) plus application shell, i18n, and integration test infrastructure — delivered in 2 days
2. **Integration test infrastructure (Story 1.2)** — Testcontainers + Respawn + pre-configured role clients became the foundation for all 9 epics. Every subsequent story inherited `IntegrationTestBase` and had real PostgreSQL tests running instantly
3. **Rigorous code review discipline** — 53 findings across 7 stories, 100% resolved. Cross-model review (different AI model for review vs implementation) caught blind spots including a critical anti-enumeration security bypass in Story 1.5
4. **Architecture compliance from story one** — ProblemDetails with `urn:sdac:*` URIs, `ServiceCollectionExtensions` as single DI entry point, snake_case naming, 9-step middleware pipeline — patterns established in 1.1 and never deviated
5. **Thorough dev notes** — Each story documented critical implementation warnings and "Previous Story Intelligence" sections, creating institutional memory that accelerated later stories

---

## Challenges

1. **Tooling version surprises on nearly every story** — Vite 7 creating vanilla TS instead of React, `.slnx` instead of `.sln`, FluentValidation v12 dropping TestHelper package, Zod v4 API changes, Testcontainers constructor deprecation. Each a small detour but they accumulated
2. **Test completeness gaps flagged in 5/7 code reviews** — Most recurring finding category. Missing cookie assertions (1.4, 1.7), missing validator tests (1.5), missing edge case coverage (1.6). Implementations worked but tests weren't proving it rigorously
3. **Code duplication in test files** — `SetUserPassword()` duplicated across 5 test files until Story 1.7 review forced extraction to `IntegrationTestBase`
4. **Resource disposal issues** — `BuildServiceProvider()` leak in 1.2 (HIGH), HttpClients never disposed in `IntegrationTestBase` (MEDIUM). Caught in review, not during implementation
5. **Rate limiter shared state** — Story 1.4: tests sharing xUnit collection shared rate limiter instance, causing test pollution. Required separate `RateLimit` collection with its own factory
6. **Password reset MVP** — Token returned in API response instead of email delivery. Accepted trade-off for internal ~30-user church app

---

## Key Insights

1. **Code review was the primary quality gate** — Almost every meaningful defect was caught in review, not in testing. The dual-model approach (one model implements, another reviews) is essential
2. **Test infrastructure investment pays compound dividends** — The ~1 story invested in `IntegrationTestBase` saved significant effort across all subsequent stories and epics
3. **Dev notes create institutional memory** — "Previous Story Intelligence" sections in story files prevented teams from repeating mistakes and provided critical context about existing code patterns
4. **Automated tests are necessary but not sufficient** — ~200+ automated tests, zero manual QA. A structured manual QA phase is critical before production
